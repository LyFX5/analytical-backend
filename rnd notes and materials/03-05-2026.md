# Preprocessing

## Data Cleaning

- enforce schema (types, units)
- remove impossible values:
    - negative fuel, km = 0 with fuel > 0
- remove deduplicates `(entity_id, year)`

## Missing Data Strategy

Split columns into:

| Type | Strategy |
| --- | --- |
| critical (fuel, km) | drop or flag |
| derived metrics | recompute |
| voluntary metrics | keep NA + flag is_missing_metric_A (Missingness itself becomes signal.) |

## Normalization

Create:

- `co2_per_km`
- `fuel_per_km`
- `km_per_year`

These become **performance axes**.

## Bucketing (for peers)

- size → bins (quantile or domain-based)
- age → bins (e.g., 0–3, 4–7, 8–12, 12+)

This stabilizes peer definitions early.

# Feature Engineering (for Clustering & Benchmarking)

## Structural features (define peers)

These should be **stable, not performance-based**:

- type (categorical → encoded)
- size_bucket
- age_bucket

## Performance features (for scoring)

- `co2_per_km`
- `fuel_per_km`
- efficiency ratios
- normalized residuals

## Temporal features

- lag features:
    - `co2_per_km_t-1`
- rolling:
    - `rolling_mean_3y`
- trend:
    - slope (linear regression per entity)

## Operator composition vector

- fleet composition:
    - % of each type
- weighted averages:
    - emissions weighted by usage

## Anomaly Detection

### Self-history anomaly

Model:

- rolling z-score
- or simple AR

anomaly = abs(x_t - rolling_mean) > k * rolling_std

### Peer-relative anomaly

Within peer group:

- compute percentile
- flag extremes:

percentile < 0.05 or > 0.95

### Combined signal

```
anomaly = (
    self_deviation_flag
    AND
    peer_deviation_flag
)
```

## Forecasting (Baseline)

### Baseline 1: Linear regression (with time + features)

- interpretable
- stable

### Option:

- **Global model (pooled regression)**

```
target: co2_per_km
features:
    age
    type
    size
    year
```

### Output

- point forecast
- confidence interval
- threshold crossing:

```
forecast > regulatory_limit
```

### Baseline 2: Gradient Boosting

- handles nonlinearity
- works well on tabular data
